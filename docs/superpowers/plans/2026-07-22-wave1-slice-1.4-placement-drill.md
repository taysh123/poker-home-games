# Wave 1 slice 1.4 — the optional placement drill

> Short slice plan. One PR: `feat/placement-drill`. TDD on all pure logic.
> Master plan §4 1.4: "5 calibrated questions offered as the FIRST recommended action after
> onboarding — best of both; skippable."

**Goal:** the self-reported skill from the funnel can be replaced by a measured one — five
questions, spread across the bank's difficulty bands, that calibrate `persona.skill`.

## Design decisions (stated, because they're honesty-adjacent)

1. **Assessment, not practice — so it is NOT metered.** The drill shows **no answers and no
   explanations** during the run: you get zero study value from it, only a level. That is what
   makes it fair to exempt from `FREE_QUIZ_PER_DAY` and the shared practice pool. (A metered
   placement would tax the user for telling us about themselves; an unmetered *practice* run
   would be a free-tier bypass. This design is neither.)
2. **Strictly one-time per account** (`persona.placement != null` hides every entry point), so
   it cannot be farmed for unmetered questions. Retaking the setup quiz does NOT clear it;
   changing skill by hand stays possible via the funnel.
3. **Honest result copy** — "a quick calibration, not a rating", and the level is always
   changeable. No score-shaming: a low score reads as "we'll start you with the fundamentals".
4. **Where it appears:** leads the funnel's router phase (first-run, the master-plan position),
   plus a quiet Study entry for anyone who skipped it. Both hidden once completed or when the
   `study` flag is off.

## Pure logic (TDD first)

`features/persona/logic/placement.ts`:
- `PLACEMENT_SPREAD = [['Beginner',2],['Intermediate',2],['Advanced',1]]` — 5 questions total.
- `placementQuestions(pool, dayKey)` — per band, take a deterministic day-seeded slice (reuses
  `dailyRotation`); if a band is thin, backfill from the remaining pool so the run is ALWAYS 5
  (or the whole pool when smaller). No duplicates.
- `skillFromPlacement(correct)` → `'new' | 'solid' | 'grinder'` (0–1 / 2–3 / 4–5; random
  guessing at 4 options ≈ 1.25 → 'new').
- `placementLevelCopy(skill)` → `{ title, body }` for the result screen (copy lives with logic).

`features/persona/types.ts`: `placement: { score: number; total: number; at: string } | null`
(optional-tolerant read for pre-1.4 stored personas).

`PersonaContext.recordPlacement(score, total)` — ONE composed commit: stores the placement AND
sets `skill` from it.

## UI

`features/persona/ui/PlacementDrillScreen.tsx` (route `PlacementDrill`, BOTH trees):
intro → 5 questions (tap an option → immediate advance, no feedback) → result card with the
level, the honest calibration line, and a primary CTA into the Spot Trainer. Skippable from the
intro and via a quiet exit; exits `goBack()`. Velvet Table tokens, reduced-motion safe,
44pt targets, a11y roles/labels, progress announced.

Entry points: OnboardingV2 router lead card ("Find your level · 5 quick questions") when
eligible; StudyScreen row when eligible.

**Non-goals:** server persona (3.2), adaptive/IRT scoring, per-category placement, harness +
screenshots (1.5).
