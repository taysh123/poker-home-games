# Wave 1 slice 1.3 — personalized surfaces

> Short slice plan. Consumers + seams verified by exploration wf_334ad99c-d40. One PR:
> `feat/personalized-surfaces`. TDD on all pure logic; screens wired on tested seams.

**Goal:** the persona actually changes the product — Home/GuestHome lead with the user's goal,
Study orders training by format, the daily quiz seeds difficulty from skill, the funnel's
promises from 1.1 become true, and the quiz is retakable (incl. by guests).

**Pure logic (TDD first):**
1. `features/persona/logic/recommendations.ts`:
   - `heroVariantForGoal(goal) → 'host' | 'improver'` — improve ⇒ improver; host/both/null ⇒ host
     (game night stays the core for 'both'; null = pre-persona default, current behavior).
   - `trainOrderForFormat(format) → TrainKey[]` — tournament ⇒ Quizzes (ICM/push-fold bank) first;
     cash/both/null ⇒ current order. Stable, complete (all 5 keys always present).
   - `difficultyForSkill(skill) → 'Beginner' | 'Intermediate' | 'Advanced' | null` — EXACT bank
     strings (verified: 708/445/307 rows); null skill ⇒ null (no filter).
2. `features/study/logic/quiz.ts` — `selectSeeded(questions, filter, seededDifficulty, minPool)`:
   applies the difficulty filter ONLY if the filtered pool ≥ minPool, else returns the
   unfiltered selection (exploration gotcha: exact-match filtering can silently empty the pool
   and shrink rotation; fallback keeps the daily quiz fresh).
