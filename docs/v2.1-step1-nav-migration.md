# V2.1 — STEP 1: Navigation IA migration notes

**Branch:** `feature/v2-poker-platform` (no PR) · **Date:** 2026-06-20 · **Reversible:** yes (new `nav5` flag, default OFF)

## Goal
Collapse the (up-to-7) tab bar into the agreed **5-tab IA**: **Home · Track · Study · Coach · Groups**. `Track` becomes a unified hub for **Bankroll + Sessions + Stats** (in-screen segments). Profile already lives in the Home header. No functionality removed; migration gated behind a flag.

## Current state (audited from `navigation/AppNavigator.tsx`)
- Two tab trees swap on auth: `TabNavigator` (authed) / `GuestTabNavigator` (guest). Both render the same 7 candidate tabs (3 flag-gated):
  `Home · AllSessions("Sessions") · Bankroll(flag) · Study(flag) · Coach(flag) · GroupsList("Groups") · Stats`.
- **Cross-tab navigations by route name (the only coupling):** `HomeScreen` → `navigate('Stats')` (L505), `navigate('AllSessions')` (L564), `navigate('GroupsList')` (L622, L673). **Nothing navigates to `'Bankroll'` by name.** No other screen jumps between tab routes.
- **Deep links / routes (`linking`):** only `JoinGroup` = `join/group/:inviteToken` and `JoinSession` = `join/session/:inviteToken`. **Neither is a tab route** → unaffected by this migration. Pending-invite resume logic (null→user transition) is untouched.
- Child screens hold their headers in isolated vars: AllSessions `stickyHeader` (title + search), Stats `customHeader` (title only), Bankroll `BrandHeader` (title + "+" add, redundant with the bottom "Log a Session" CTA), LocalSessions/GuestStats (inline title + `insets.top`).

## Approach (additive + flag-gated)
1. **New flag `nav5`** (`config/features.ts`): PROD **off** (prod = today, byte-identical), DEV **on** (preview). Single switch; reversible.
2. **`TrackScreen` hub** (`screens/TrackScreen.tsx`): owns a "Track" header + a segmented control. Reads `useAuth()` to pick children:
   - authed: `BankrollScreen` (if `bankroll`) · `AllSessionsScreen` · `StatsScreen`
   - guest: `BankrollScreen` (if `bankroll`) · `LocalSessionsScreen` · `GuestStatsScreen`
   - Accepts `route.params.segment` to preselect; conditionally renders only the active child (remount → its load/focus effects fire normally).
3. **Additive `embedded?: boolean` prop** on the 5 child screens — when true, hides the redundant screen **title** and drops the top **safe-area inset** (the hub provides both). All functional sub-elements (search, period picker, add via bottom CTA) are preserved. **Default `false` ⇒ standalone render unchanged.**
4. **`navHelpers.ts`**: `goToSessions(nav)` / `goToStats(nav)` route to `Track{segment}` when `nav5` is on, else to the legacy `AllSessions`/`Stats` tabs. `HomeScreen` uses these (Groups nav unchanged — `GroupsList` survives in both IAs).
5. **`AppNavigator`**: when `nav5` on, both tab navigators render **Home · Track · Study(flag) · Coach(flag) · Groups** (≤5). When off, the existing tabs render exactly as today. `Track` added to `TabParamList` + `RootStackParamList`.

## Backward-compat & route verification
- `AllSessions`/`Stats`/`Bankroll` remain valid components — under `nav5` they render **embedded inside Track** (not removed). Under `nav5` off they remain standalone tabs.
- The only by-name cross-nav (HomeScreen) is routed through `navHelpers`, so it resolves correctly in **both** IAs.
- Deep links (`join/*`) and the invite-resume flow are independent of tabs → verified unaffected.
- `LiveGameBar`, push listeners, modals (Login/Register/Paywall/LogSession) unchanged.

## Risks & mitigations
| Risk | Mitigation |
|------|-----------|
| Embedded child double-renders header / safe area | `embedded` drops title + top inset; verified per-screen |
| Bankroll "+" add lost when its header hidden | Bottom "Log a Session" CTA already covers it (no loss) |
| Segment switch loses scroll/state | Acceptable for V2.1 (remount reloads); documented; can memoize later |
| Name-nav to a removed tab under `nav5` | Only HomeScreen cross-navs; all routed via `navHelpers` |
| Prod regression | `nav5` defaults OFF → prod render path is unchanged |

## Rollback
Flip `nav5` off (or remove the DEV override). No data, route, or deep-link changes to revert; `embedded` defaults keep child screens identical.

## Verification plan
`npx tsc --noEmit` + `npx jest` (regression) + manual smoke: nav5 OFF (today's 7-tab unchanged) and nav5 ON (5 tabs; Track segments switch; Home "See all sessions/stats" land on the right segment; deep-link join still resolves; LiveGameBar shows).
